A analysis of commit history for files from Plone Python CMS

Enoque Joseneas



Plone is a free and open source content management system built on top of the Zope application server. Plone is positioned as an "Enterprise CMS" and is most commonly used for intranets and as part of the web presence of large organizations. High-profile public sector users include the U.S. Federal Bureau of Investigation, Brazilian Government, United Nations, City of Bern (Switzerland), New South Wales Government (Australia), and European Environment Agency. Plone's proponents cite its security track record and its accessibility as reasons to choose Plone. This work aimed to analyze the history of commits per file during the last 9 years of Plone Python CMS (2008~2017). The Qt toolkit was used to consume the github API via http requests and to build the directory tree and files using a Composite design pattern implementation. For the construction of the database, the github REST service was used to read information from the directory tree and files and also, the commits from the last 9 years. To construct the visualization, an algorithm was developed that extracted the information of commits by file generating a json file, mapped as a HashMap: file_absolute_path -> commits_list. After this step, a second file containing the structure of the molecules was builded, and the following information was used for each file: the total changes (made in the file for each commit), the date the commit was performed, the name of commiter, the file name and the blob url in github. After the analysis of the visualization, it was observed that plone has a relatively small number of changes per file, which confirms its focus on stability, security and, consequently, fewer bugs. In addition, it has also been noted that Plone has a poor organization of directory and file structure.
